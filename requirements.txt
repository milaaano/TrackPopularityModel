# WHY EVERYTHING IS PINNED WITH ==, NOT >=
# The files in model/artifacts/ are *pickles*. Unpickling a scikit-learn estimator
# under a different sklearn version raises InconsistentVersionWarning and is not
# guaranteed to reconstruct the same object — it can load "successfully" and then
# predict differently. A wrong-but-plausible score is the failure we can't see, so
# the serving env must match the training env exactly. If you upgrade sklearn or
# lightgbm, RETRAIN and regenerate the artifacts; do not just bump the pin.
#
#   python -m venv .venv && source .venv/bin/activate
#   pip install -r requirements.txt
